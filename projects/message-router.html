Iceberg

Message Router

Iceberg is a highly modular and fault tolerant message routing platform designed to meet high scalability and availability requirement. This platform horizontally scales your application. See Example. Iceberg is used to send messages from source to destination asynchronously

Technologies used

  • Java
  • Spring
  • RabbitMQ
  • MySQL

Message Router Components

Router

Router is a REST service which gives endpoint to send message. It contains routing logic to route message among queues. This routing logic is configurable in routing table.

Processor

Processor is a console application which connects to queue and dequeue messages. It takes service_id in command line parameter, which is used to get connection data from connection table. Using this connection data, processor dynamically connects to a queue. Processor also has a MessageHandler interface.
